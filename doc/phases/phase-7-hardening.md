# Phase 7: Cross-Cutting Hardening

## Objective
Harden the application against security vulnerabilities, establish input validation boundaries, rate limiters, and set up OpenAPI documentation.

## Scope
*   **In Scope:**
    *   Enforce request input DTO validations globally using NestJS `ValidationPipe` with strict properties settings (`whitelist: true`, `forbidNonWhitelisted: true`).
    *   Integrate CORS controls specifying permitted origins.
    *   Add security headers using `helmet` middleware.
    *   Configure API rate limiting via `@nestjs/throttler` with a Redis backend to support distributed state.
    *   Add a request tracing interceptor to generate and inject unique correlation IDs (`x-request-id`) into logging parameters.
    *   Clean/sanitize text inputs (removing dangerous HTML/script injections).
    *   Set up OpenAPI schemas (`@nestjs/swagger`) to generate interactive documentation at `/v1/docs`.
*   **Out Scope:**
    *   Configuring production firewall filters or server load balancers.

## Dependencies / Entry Criteria
- Features from Phase 2 through Phase 6 functionally complete.
- Redis server is available for throttler state storage.

## Folder Structure
```text
src/
├── main.ts                              # Configuration for Helmet, CORS, Throttlers, and Swagger
├── shared/
│   ├── interceptors/
│   │   └── trace.interceptor.ts         # Generates x-request-id traces
│   └── guards/
│       └── rate-limit.guard.ts          # Throttler custom overrides
```

## Endpoints & Entities Touched
- **Endpoints:**
  - `GET /v1/docs` (OpenAPI Swagger Documentation page)
  - All functional endpoints (now wrapped in rate limiters and validation pipes)
- **Entities:**
  - None

## Acceptance Criteria
- [ ] Submitting request bodies containing undocumented fields is blocked and returns an HTTP 400 validation envelope.
- [ ] Exceeding defined throttler boundaries (e.g., more than 60 calls per minute on public routes) returns an HTTP 429 Too Many Requests response.
- [ ] Accessing `/v1/docs` displays the Swagger UI mapping all endpoints, expected parameters, and HTTP responses.
- [ ] Headers include protection keys (`X-Frame-Options`, `Content-Security-Policy`) generated by Helmet.
- [ ] Log outputs contain the unique `requestId` tracing key for every inbound route execution.

## Risks & Open Questions
- Staged configurations must differentiate rates: sensitive auth endpoints (login/register) must have significantly tighter rate limit boundaries than public lookups (cities listing).
